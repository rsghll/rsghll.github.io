FastDFS 介绍

  • fastDFS 是以C语言开发的一项开源轻量级分布式文件系统，
  • 他对文件进行管理，主要功能有：文件存储，文件同步，文件访问（文件上传/下载）,特别适合以文件为载体的在线服务，如图片网站，视频网站等

  • 为互联网量身定制，充分考虑了冗余备份，负载均衡，线性扩展等机制，并注重高可用，高性能等指标，

  • 可以帮助我们搭建一套高性能的文件服务器集群，并提供文件上传，下载等服务。

FastDFS 分为:

​ Client ：

​ 客户端：可以是浏览器，docker容器，django 服务

​ Tracker 群

​ + 监控管理 Storage群

​ Storage 群

          - 真正存储文件的地方
                - Volume 称为一卷或一组，主从同步差不多，不是主从的概念，都可以写读，但是一个写，会同步给其他节点，

单个卷的作用：从读写和容量来考虑，读取性能变高没，写入的性能变高没，容量变大没

  • 一个卷 写入并没有变高，写一个文件，另一个机器必须的在写一次
  • 读的性能会变高，可以一次读取十个
  • 容量没有变大，以为会重复文件，文件都一样

Storage 集群

  • 写入卷一和卷二，文件被分片，就会加强写入和容量，读取也被扩展，
  • 动态扩展，高可用（卷一机器坏了，还有其他机器），高性能，可扩展

客户端：

  • （写入那个，读那个）没有核心节点， 直接跟卷里的某个机器读写，（先确定机器）
  • 首先先确定跟那个机器读写，必须跟Traker 群（交互）确定，
  • ​ 客户要读那个文件，Tracker（与集群有交互，知道集群在干嘛） 就告诉你取第二个机器去读，
  • Tracker 看谁闲，告诉你你就找谁

fastdfs集群:

​ 注意: 一定要删除 /var/fdfs/ 目录下的文件 (此目录下的文件包含旧的配置, 会造成服务无法访问)

# 加载docker镜像
sudo docker load -i fastdfs_docker.tar

# 启动tracker
sudo docker run -dit --name tracker --network=host -v /var/fdfs/tracker:/var/fdfs delron/fastdfs tracker

# 启动storage
sudo docker run -dti --name storage --network=host -e TRACKER_SERVER=10.211.55.15:22122 -v /var/fdfs/storage:/var/fdfs delron/fastdfs storage

# http://10.211.55.15:8888/  这是 storage的访问地址

# 自定义存储后端
# utils/fdfs/my_storage.py
from django.core.files.storage import Storage
from django.conf import settings
from fdfs_client.client import Fdfs_client, get_tracker_conf


class MyStorage(Storage):
    def __init__(self):
        self.base_url = settings.FDFS_BASE_URL

    def _open(self, name, mode='rb'):
        pass

    def _save(self, name, content):
        # 返回 真实存储的 文件名
        # 1. 读取配置文件
        conf = get_tracker_conf(conf_path=settings.FDFS_CONF_PATH)
        # 2. 创建FastDFS客户端实例
        client = Fdfs_client(conf)
        # 3. 调用FastDFS客户端上传文件方法
        ret = client.upload_by_buffer(content.file.read())
        return ret['Remote file_id'].decode()

    def url(self, name):
        return self.base_url + name

    def exists(self, name):
        return False
